## 0.1.0 Scope: App shell + data model + settings

### What was built

1. **SvelteKit project scaffolding** (`app/` directory)
   - SvelteKit 2 + Svelte 5, TypeScript, Tailwind CSS 4, Vite
   - Static adapter for GitHub Pages
   - `@vite-pwa/sveltekit` for Service Worker + installability
   - ESLint + Prettier (tabs, single quotes, no semicolons)
   - Vitest setup

2. **Data model** (`app/src/lib/models/types.ts`)
   - TypeScript interfaces: `Vineyard` (with `row_count`, `direction_label`), `Scan`, `RowVideo`, `VineMap`, `BbchResult`
   - `DirectionLabel` type: `top_bottom | left_right` (per-vineyard config)
   - Video status type: `LOCAL_ONLY | UPLOADING | PROCESSING | DONE | FAILED`
   - Settings type: `github_repo`, `github_pat`, `google_api_key`

3. **Storage layer** (`app/src/lib/storage/idb.ts`)
   - IndexedDB wrapper using `idb-keyval`
   - Functions: save/load/delete/list for Vineyard
   - Settings persistence (credentials in IndexedDB)

4. **Stores** (`app/src/lib/stores/`)
   - `vineyard.ts` вЂ” Svelte store for vineyard CRUD, current vineyard selection
   - `settings.ts` вЂ” Svelte store for credentials (load/save from IndexedDB)

5. **Routes / screens**
   - `+layout.svelte` вЂ” app shell with header, vineyard selector, nav, offline indicator
   - `+page.svelte` (Home) вЂ” vineyard selector, scan list, new scan button, settings gear
   - `/settings/+page.svelte` вЂ” form for GitHub repo URL, PAT, Google API key; delete vineyard
   - `/vineyard/new/+page.svelte` вЂ” create vineyard form (name, row count, walking direction)

6. **Unit tests**
   - Data model validation
   - Storage layer with `fake-indexeddb`
   - Vineyard CRUD operations
